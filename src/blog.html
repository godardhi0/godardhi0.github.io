• Integration is one of many beautiful operations in math.
    So i wrote a C code that calculates the defined integral of linear and polynomial functions. Its not the most efficient but it's a start.

    Definition: Suppose \(\mathcal{f}\) is continuous. A left Riemann sum of \(\mathcal{f}\) of order \( \mathcal{n}\) on an interval [a, b] is number given by :

    \(S_n (\mathcal{f}[a,b]) := \frac{b-a}{n} \sum_{k=0}^{n-1}\mathcal{f}(a+\frac{k}{n}(b-a))\).

    The right sum is obtained by summing \( \mathcal{k} \) from 1 to \( \mathcal{n}\).

    The Riemann sum defined above is the integral of a step function equal to \( \mathcal{f}(x_{k-1,n}) \) (resp. \( \mathcal{f}(x_{k,n}) \) for the right Riemann sum)
    over every interval ]\( x_{k-1,n},x_{k,n}\)[ with \( x_{k,n}:= a+k\frac{b-a}{n}\) for \( \mathcal{k \in}\) [0,n].

    So for every continous function \(\mathcal{f}\) on [a, b] the sequence \( S_n(\mathcal{f}[a, b])\) converges to the integral of \(\mathcal{f}\) as \( n \) gets larger.
    code: 
                                
                                for i=1 to n do
                                    x <- a + i(b-a)/n
                                    right_sum <- right_sum + (b-a)/n*f(x)                                     
                                END for
                                    RETURN right_sum
    Now the next task it to express \(\mathcal{f}\) in code. Two of the most common types of functions are linear and non-linear functions.
    A linear maping is just a function Double\(\mathcal{\_linear}\)(const double slope, double x, const double b) that returns a number \(\mathcal{y}\) such that \(\mathcal{y}=slope*x + b\)
    A polynomial is a type of non-linear function. Polynomials are defined as sums of products :

    \(\sum_{i=0}^{N}\mathcal{a_{i}X^{i}}\).

    where \(\mathcal{X}\) is the varaible and \(\mathcal{a_{i}}\) is ith coeficient. So the expression of a polynomial function in code could be Double \(\mathcal{\_poly}\)(const double *coef_degre, double variable, const int coef_degre_size).
    If \(\mathcal{y}\) is the number returned by \(\mathcal{\_poly}\) then \(\mathcal{y}\) should be obtained with some sort of loop that adds all the terms of the polynomial.
